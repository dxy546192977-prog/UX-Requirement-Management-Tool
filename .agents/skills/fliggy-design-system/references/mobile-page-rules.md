# 普通移动端页面规则

## 核心原则

- 页面首先是移动端交易界面，不是运营海报
- 先保证信息效率，再补品牌氛围
- 统一版心、统一间距、统一圆角层级
- 页面骨架优先和业务目标一致，而不是沿用固定模板
- 图片、标题、筛选、价格、CTA 都要服务当前业务语义

## 按钮规则

- 普通页面的常规主按钮默认使用黄底黑字
- `预订`、`立即抢`、`支付` 这三类强交易动作优先使用红底白字
- 同一页面如果同时存在普通操作和强交易操作，按钮颜色必须拉开层级，不要全部做成同色
- 次级按钮、筛选按钮和辅助操作优先使用浅底或描边，不要抢主交易 CTA

## 常用骨架

- 频道页：导航 + 主题入口 + 混合信息流
- 列表页：条件条 + 筛选条 + 连续列表
- 详情页：摘要区 + 选择区 + 说明区 + 吸底 CTA
- 下单页：订单摘要 + 表单区 + 价格区 + 支付区 + 提交按钮

## 场景区别

- 频道页更强调主题、内容分发和混排
- 列表页更强调筛选效率、摘要信息和连续浏览
- 详情页更强调商品摘要、权益解释和购买承接
- 下单页更强调确认、填写和提交，不要继续做导购

## 版式规范

- 默认以 375 逻辑宽组织内容
- 内容区左右边距保持一致
- 吸底区、tabbar、输入区要提前预留空间
- 卡片内容必须在一屏逻辑宽内完整成立，不允许视觉溢出
- 卡片之间优先使用 12 / 16 的节奏，不要忽大忽小

## 下单页重点

- 先展示已选商品摘要
- 顶部优先使用黄色品牌氛围和简洁导航，不要把详情页大图头部直接搬到下单页
- 再给房型、日期、入住人等必要决策项
- 表单字段要能即时校验
- 价格明细要拆解，不只显示总价
- 支付方式要有选中、禁用、推荐等状态
- 提交按钮要固定可见，但不能遮挡内容
- 可补充流程服务条、出行人卡、联系人卡、规则提示卡等确认型模块
- 下单按钮文案要清楚表达动作，如确认订单、去付款、继续预订
- 如果参考页面顶部只有品牌色氛围，不要私自补酒店详情图

## 状态要求

- 默认态
- 加载态
- 错误态
- 空态
- 禁用态
- 字段级报错态

## 避免事项

- 不要把详情页大图和营销氛围直接搬到列表页或下单页
- 不要让筛选 chips 和卡片重复表达同样信息
- 不要让 fixed CTA 压住卡片内容或表单报错
