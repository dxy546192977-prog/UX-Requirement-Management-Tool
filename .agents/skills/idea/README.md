# Idea-fusion

融合四大脑暴 Skill 的核心机制，从模糊方向走到清晰的 Idea Brief。

一个对话式想法引擎——不是帮你优化你已经有的答案，而是帮你找到你还没想到的问题。

---

## 这个 Skill 做什么

**核心价值：** 在你锁定方向之前，把可能性空间系统打开——四个视角发散、对抗性过滤、收敛对话——最终产出一份经过压力测试的 Idea Brief，而不是你第一个直觉的精装版。

**解决什么问题：**
- 想法还没想透就进入执行，方向选错了，后面越做越偏离
- 独自思考时停在显而易见的第一层，看不到自己跳过了什么
- AI生成选项，用户只是在打分，真正的判断从未发生

**适用场景：**

| 场景 | 你会得到什么 |
|------|-------------|
| 新功能 / 新产品方向探索 | Grounding 挖掘真实约束、四个视角发散候选方向、过滤后的幸存者列表、收敛到 Idea Brief |
| 有想法但不确定问题对不对 | Pressure Test 质疑目标本身、代理目标识别、快速路径直达收敛 |
| 非产品类创意——品牌、内容、空间 | Universal 模式按你指定领域运行同一套发散-收敛逻辑 |

**不做的事：**
- 你已有清晰方向需要审查 → 用 `/ceo`
- 你需要需求文档或技术方案 → 这里产出的是 Idea Brief，不是 PRD

---

## 快速上手

在 IDE 中输入 `/idea`，然后描述你想探索的方向。不需要想好再开始——模糊的起点是正常的起点。

**最佳实践：**

- **先说问题，不要先说解法。** Grounding 阶段会帮你挖清楚真实约束，过早锁定解法会让发散失效
- **推理主权阶段认真回答"你已经想过什么"。** 哪怕只是粗糙的直觉——你说得越具体，发散结果和你的判断框架对齐得越好
- **对抗性过滤阶段不要护着你喜欢的想法。** 能扛住拒绝准则的才值得继续，被拒的写了原因，随时可以翻案
- **Brief 产出后直接接 `/ceo`。** 带着一份经过压力测试的方向做商业审查，比带着直觉去要扎实得多

---

## 核心流程

```
Scope Gate → Grounding → Pressure Test → 用户已有想法
    → Divergence → Adversarial Filter → Survivor → Convergence → Idea Brief
```

每个阶段都有明确的退出条件。快速路径（需求已清晰）可以跳过 Divergence 直达 Convergence。

---

## 推理主权

**为什么不一上来就直接发散**

AI 先生成，你的认知角色就从「建构」变成「评估」。锚定效应会让 AI 的第一个框架主导整个会话——你以为你在独立判断，实际上是在对 AI 的坐标系做微调。

所以流程强制先问：

> “在我开始生成想法之前——你已经想过哪些方向，或排除了哪些？粗略的直觉也算。”

目的只有一个：让你的推理在 AI 介入前先成形。

## 三种模式

| 模式 | 你的状态 | 你会得到什么 |
|------|---------|-------------|
| **Venture** | 有明确落地目标，要服务真实用户 | 在约束下发散，把需求想透再收敛 |
| **Builder** | 好奇心驱动，还没想清楚做成什么样 | 打开可能性空间，发现自己没想到的角度 |
| **Universal** | 非数字产品——品牌、内容、空间等 | 按你指定的领域套用同样的发散-收敛逻辑 |

不确定选哪个？直接描述你的情况，Skill 会根据你说的内容自动判断最适合的模式，进入对话后随时可以切换。

## 发散引擎：四个视角

不是随机生成，是从四个不同的认知框架系统发散，确保覆盖你单独思考时容易跳过的盲区：

| 视角 | 问的问题 |
|------|------|
| **用户摩擦视角** | 用户现在在忍受什么？那个摩擦背后是什么？ |
| **消除/反转视角** | 如果把这个东西彻底去掉或反过来会怎样？ |
| **假设挑战视角** | 这个方向里有哪些没人质疑的前提？它们都对吗？ |
| **杠杆/复利视角** | 什么样的动作会产生不成比例的长期回报？ |

四个视角结束后做跨视角合成——找那些在多个视角里都有信号的方向，那里往往藏着真正有意思的东西。

---

## 对抗性过滤

不是帮你从选项里挑一个，而是让每个想法证明“自己值得留下来。

每个生成的想法必须通过拒绝准则——满足任一就出局：

- **模糊**：无法在一句话内说明具体机制
- **重复**：与另一个想法在机制上无区别
- **不可行**：在已知约束下无法执行
- **情绪空洞**：功能正确但用户不会有任何感觉
- **被取代**：在目标用户场景下，现有方案已经解决得很好，且这个想法没有说清楚增量优势在哪

每个被拒绝的想法都会写明原因。静默拒绝是不允许的。

## 产出物

落盘路径：`sessions/{topic-slug}/`

### Idea Brief（自动落盘）

一份结构化的想法简报，不是执行计划，不是需求文档：

```
- 选定方向 + 为什么选它
- 范围边界（In / Out）
- 工作假设 vs 已确认事实
- 问题澄清记录（改变了方向的关键问答）
- 开放问题
- 下一步实验（验证动作，不是开发步骤）
```

### 对话记录（自动落盘）

生成完整对话过程的 HTML，方便人类阅读、沟通和交流。

## 适合衔接的工作流

```
/idea → Idea Brief → /ceo → 商业审查 → 执行规划
```

`/idea` 产出的 Idea Brief 直接可以作为 `/ceo` 的输入——带着一份经过压力测试的方向去接受 CEO 级的商业审查，比带着第一个直觉去审查要扎实得多。

## 安装

### IDE Skill（手动）

| 平台 | 安装方式 |
|------|----------|
| Qoder | 复制目录到 `~/.qoder/skills/idea/` |
| Cursor | 复制目录到 `.cursor/skills/idea/` |
| Windsurf | 复制目录到 `.windsurf/skills/idea/` |
| Codex | 整个目录放入 skills 目录 |
| Claude Code | 贴到 `CLAUDE.md` 或 `.claude/commands/idea/` |

### LLM 直接使用

把 `SKILL.md` 丢到 GPT / Gemini / Claude 对话框：

```
使用这个文件作为你的思考框架和工作流程
```

然后直接描述你想探索的方向。
