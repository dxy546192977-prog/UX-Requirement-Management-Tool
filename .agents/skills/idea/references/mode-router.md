# Mode Router

idea 的模式路由逻辑。SKILL.md Phase 0.2 引用本文件。

---

## 三种模式

### Venture 模式

**适用场景：** 有商业意图的产品或服务想法。用户提到客户、收入、融资、竞品、市场。

**核心姿态：** 需求真实性压力。默认假设：用户的自我叙事可能偏离现实。

#### 六问认知骨架

Venture 模式的默认对话骨架。顺序是认知依赖链，不是风格选择：需求真实性必须在切口选择之前确认，现状必须在未来适配之前确认。

| 序号 | 认知焦点 | 核心问题（重写版，非 YC 原文） |
|------|---------|---------------------------|
| Q1 | 需求真实性 | 谁真正需要这个？你看到了什么让你相信这个需求是真实的？ |
| Q2 | 现状分析 | 现在这些人怎么解决这个问题？为什么现有方案不够好？ |
| Q3 | 具体用户 | 描述一个你最清楚的目标用户——他的场景、痛点、决策过程 |
| Q4 | 最小切口 | 如果只能解决一个问题，是哪个？为什么从这里切入？ |
| Q5 | 观察与惊喜 | 在你了解这个领域的过程中，什么发现让你意外？ |
| Q6 | 未来适配 | 如果成功了，3 年后这个方向长成什么样？ |

#### 按产品阶段 Smart-Skip

不是所有六问都对每个阶段有用。根据产品阶段选择性运行：

| 产品阶段 | 运行的问题 | 跳过理由 |
|---------|-----------|---------|
| **Pre-product**（还没产品） | Q1, Q2, Q3 | 没有用户和切口前，Q4-Q6 是空想 |
| **Has users**（有用户了） | Q2, Q4, Q5 | 需求已验证，聚焦切口和反直觉发现 |
| **Paying customers**（有付费用户） | Q4, Q5, Q6 | 基础已稳，聚焦扩展和长期适配 |

完整的六问序列仍是默认。Smart-skip 需要产品阶段信息才能激活。如果阶段不明，跑完整序列。

用户可以明确要求重新排序。AI 不得自行重排。

#### 两条反自我叙事原则

**原则 1：用户的话比创始人的叙事重要。**

> 创始人说的"产品做什么"和用户说的"产品做什么"之间几乎总有差距。用户的版本是真相。

在 Venture 模式中，Skill 有风险接受用户对自己产品的叙事而不测试它。当用户描述产品时，问：

> "你的实际用户怎么描述你的产品为他们做了什么？"

如果描述和用户的框架不同，明确标记这个差距。

**原则 2：观察优先于演示。**

原始规则是"去看真实用户操作"。在没有原型的 ideation 场景中，不能字面执行。编码为引导规则：

> 当用户做出以"我觉得用户会……"或"用户大概会……"开头的断言时，回应：
> "你有没有看过一个真实用户在没有引导的情况下面对这个场景？他们做了什么？"

这把观察原则转化为可操作的引导动作，检验用户关于用户行为的断言是基于观察还是投射。

### Builder 模式

**适用场景：** 个人项目、学习实验、开源、黑客松、纯粹的好奇心驱动。

**核心姿态：** 热情驱动、可能性优先。追问力度低于 Venture，重点在发散和可能性空间。

不使用六问认知骨架。由 Grounding Packet + Divergence Engine 的标准流程驱动。

### Universal 模式

**适用场景：** 非软件领域——内容策略、运营方案、组织设计、教育课程等。

**核心姿态：** 灵活适配、去技术假设。不预设问题域是软件产品。

不使用六问认知骨架。由 Grounding Packet + Divergence Engine 的标准流程驱动。

## 跨模式检测规则

以下规则不属于任何单一模式，而是在所有模式中持续监测。

### 多利益相关者对齐

**触发信号（满足任一）：**
- 用户在描述目标时用"我们"（不是修辞性的"我们"）
- 用户提到联合创始人、合伙人、或偏好不同的团队成员
- 用户说需要说服 VP、赞助人或决策者
- 用户描述多个利益相关者之间的优先级冲突

**当触发时：** 会话框架从"帮你做决定"变为"帮你找到对齐点"。在生成想法之前，询问每个利益相关者的优先级以及分歧所在。

### 中途模式升级

持续监测。在 Grounding 和 Convergence 阶段，如果检测到模式信号变化：

- **Builder → Venture：** 用户提到客户、收入、融资
  > "听起来这不只是个 side project——你提到了 [信号]。我切到 Venture 模式，会用更严格的需求真实性检验。可以吗？"

升级需要用户确认。不要静默切换。
