# Convergence Engine

idea 的收敛引擎。包含 Survivor Presentation 规则和 Layer 1/2/3 景观推理。
SKILL.md Phase 5 和 Phase 6 引用本文件。

## Survivor Presentation 规则

Adversarial Filter 之后、Convergence Dialogue 之前的关键展示阶段。

### Rule A: 先展示全部，推荐放最后

当展示幸存想法时，先展示所有想法及其理由，然后再命名推荐方向。

> 推荐是最后说的东西，不是第一个说的。

如果 AI 的推荐出现在第一位，用户在独立评估其他选项之前就已经被锚定到那个偏好了。

**这条规则在所有模式和视觉格式下都适用。**

每个幸存想法的展示格式：
1. **想法名称** —— 一个直觉可理解的短标签
2. **核心机制** —— 一句话说清楚它怎么运作
3. **为什么它活下来了** —— 在 Adversarial Filter 中幸存的原因
4. **风险 / 开放问题** —— 最大的不确定性是什么

展示完全部之后，再给推荐：
> "如果要我选一个最值得深入探索的方向，我会选 [X]，因为 [一句话原因]。"

### Rule B: 轻量视觉格式决策

完整的 Visual Companion（浏览器渲染、每问一个 HTML mockup）不适用于纯 ideation Skill。
核心原则以精简形式保留。

当展示 2 个以上幸存想法进行对比时，做一个展示判断：
视觉并排（对比表或结构图）是否会比纯文字让对比显著更清晰？

**触发条件（满足任一即触发）：**
- 幸存想法的核心差异是结构性或空间性的（如不同的导航模式、产品范围边界、信息层级）
- 用户是非设计背景，对比涉及产品结构或体验形态

**当触发时，在展示前询问：**
> "你更想看并排对比表，还是文字描述？"

**当没有触发条件时：** 默认使用文字，不询问。

## Layer 1 / 2 / 3 景观推理

在 Convergence Dialogue 开始前，做显式的三层思考。这不是姿态，而是一个可执行的思考序列：

### Layer 1: 传统智慧

> 这个领域里所有人都已经知道的是什么？

列出公认的行业常识、最佳实践、主流方法论。这是基线。

### Layer 2: 当前信号

> 当前的案例和话语在说什么？

最新的成功案例、失败教训、行业趋势、用户行为变化。这是现实校准。

### Layer 3: 第一性原理

> 根据本次会话中涌现的信息，传统智慧在这里是否错了？

用会话中收集到的具体信息（Grounding Packet、用户回答、Divergence 输出）挑战 Layer 1 的常识。

### Eureka 检查

> 当第一性原理推理推翻传统智慧时——明确命名这个发现。

这是 Convergence 阶段能产出的最高价值输出。不要把它埋在总结里。单独标出：

> "**Eureka：** [传统智慧] 说 X，但本次会话的证据表明 Y。如果 Y 成立，这改变了 [什么]。"

如果没有 eureka 发现，不要编造。直接进入 Convergence Dialogue。
