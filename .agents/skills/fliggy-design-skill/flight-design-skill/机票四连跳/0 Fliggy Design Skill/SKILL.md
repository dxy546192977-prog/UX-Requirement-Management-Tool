# Fliggy Design Chat

飞猪对话式 AI 旅行助手的移动端 UI 设计规范，指导大模型输出符合飞猪设计标准的 HTML + CSS 代码。
![效果预览](https://gw.alicdn.com/imgextra/i1/O1CN01s0RrkS1PoIVYVR1Nq_!!6000000001887-2-tps-1302-4854.png)

> **读取策略（重要）**：本文件是唯一入口，包含所有决策所需的信息。**禁止一次性读取全部组件文件。** 正确流程：读完本文件 → 需求分析 → 确定需要的组件 → 仅读取对应的组件 README → 生成代码。example.html 仅在组件结构不明确时才查阅。

## 一、技术基准

- 输出格式：单文件 HTML + 内联 CSS（`<style>` 标签）
- 目标平台：移动端（App WebView / H5）
- 设计基准：750 × 1624px（@2x），所有组件标注均为 @2x px 值
- **适配方案**：通过固定 viewport 宽度实现等比缩放，`<head>` 中必须包含：
  ```html
  <meta name="viewport" content="width=750, user-scalable=no">
  ```
  原理：浏览器将布局视口锁定为 750px，在不同宽度的设备上自动等比缩放（375px 设备 → 缩放 0.5×，414px → 0.552×）。因此所有组件直接使用 @2x px 值书写，无需做任何单位换算。
- 页面边距：30px，间距体系遵循 6 的倍数栅格（6/12/18/24/30/36/42/48px）
- 模块垂直间距由各模块自身 margin-top 控制，不使用全局间距 token
- AI 回复不输出头像和名称标识，直接呈现对话内容

## 二、Design Tokens（全局规范，始终生效）

### 字体

```css
@font-face {
  font-family: 'Fliggy Sans 102';
  src: url('https://g.alicdn.com/trip/common-assets/1.1.9/fonts/FliggySans102-Md.ttf') format('truetype');
  font-weight: 500;
  font-display: swap;
}
```

- 数字/货币字体：`font-family: 'Fliggy Sans 102', 'PingFang SC', 'SimHei', sans-serif; font-weight: bold;`
- 默认汉字字体：`font-family: 'PingFang SC', 'SimHei', sans-serif;`

### 颜色

```css
:root {
  --color-darkgray: #0f131a;   /* 标题、正文、价格数字 */
  --color-midgray: #5c5f66;    /* 引用、副标题 */
  --color-lightgray: #919499;  /* 标签文本 */
  --color-indigo-1: #6666ff;   /* 品牌主色：仅限按钮文本和链接，禁止大面积使用 */
  --color-indigo-4: #EBEBFF;   /* 品牌辅助色：按钮背景 */
  --color-warning-1: #ff3333;  /* 警示色 */
  --color-pay-1: #ff5533;      /* 优惠色 */
  --color-white: #ffffff;      /* 卡片背景 */
  --color-label: #F7F8FA;      /* 全局页面背景色 */
  --color-bg: #f2f3f5;         /* 分割线、次级按钮背景 */
}
```

### 圆角

```css
:root {
  --radius-xl: 36px;  /* 大卡片（交通卡、商品卡、酒店房型卡、下单卡、订单卡、表格等） */
  --radius-l: 24px;   /* 小卡片、图片容器、按钮、标签 */
  --radius-m: 12px;   /* 小元素、内嵌缩略图 */
  --radius-s: 6px;    /* 极小元素、进度条、色块角标 */
}
```

### 文本排版规则

Markdown 文本直接平铺在页面背景（#F7F8FA）上，**禁止**用白底卡片包裹文本。白底卡片仅用于固定组件（交通卡、商品卡等）。

## 三、需求分析（每次生成前执行）

收到需求后，按以下步骤分析：

1. **场景识别**：搜索推荐？行程规划？预订下单？订单查看？
2. **对话流设计**：用户输入 → AI 回复包含哪些内容类型？
3. **组件选择**：查阅下方「组件索引」，确定需要的组件列表
4. **按需读取**：仅读取选中组件的 README.md，不读取其他文件

分析结果格式：
```
【需求摘要】一句话描述
【核心模块】需要的组件列表（从索引中选择）
【对话流】用户输入 -> AI 回复的流程
```

## 五、排版与输出策略

### 方案对比模式（搜索/推荐场景）

多方案并列，每个方案 = H2 标题 + 简要文本 + 固定组件卡片。交通类固定 3 方案（如"耗时最短"/"高性价比"/"均衡之选"）；酒店类按区域分组每组 2-3 家；行程规划按天或方案分组。

### 渐进式对话流（预订场景）

信息展示 → 用户确认意向 → 下单卡。先输出信息（文本+图片+商品卡），等用户明确购买意图后再出下单卡。例外：用户需求已非常明确时可直接出下单卡。

### 组件选择优先级

| 业务实体 | 首选组件 |
|----------|----------|
| 机票/火车/汽车 | 交通卡 |
| 酒店 | 商品卡（需展示房型时用酒店房型卡） |
| 景点/POI/门票/签证/套餐 | 商品卡 |
| 下单确认 | 下单卡 |
| 订单详情/退款 | 订单卡 |
| 用户选择 | 选择器 |

### 输出量控制

交通推荐最多 3 方案，酒店每区域 2-3 家，行程 2-3 方案。每方案文本 2-3 句。超出部分用扩展按钮收起。

## 六、输出规范

- 所有样式使用 `<style>` 标签，不依赖外部样式文件
- **颜色和圆角必须使用 `var(--token-name)` 引用**，禁止硬编码 hex 值或 px 数值。唯一的 token 定义来源是本文件「二、Design Tokens」中的 `:root` 声明
- 生成的 HTML 必须在 `<style>` 标签中包含完整的 `:root` 变量声明（颜色 + 圆角），确保 `var()` 引用能正确解析
- 品牌色 `var(--color-indigo-1)` 仅限按钮文本和链接，禁止用作卡片底色、icon、大面积填充
- 图片使用 Unsplash 免费图片服务填充，详见下方「八、图片填充规范」
- 绝不允许出现规范之外的色值、间距或圆角

## 七、Token 维护指南

当需要批量修改设计风格时（如品牌升级、主题切换），按以下步骤操作：

1. **修改本文件**「二、Design Tokens」中 `:root` 里的变量值（如 `--color-indigo-1: #6666ff;` → `--color-indigo-1: #7777ff;`）
2. **同步修改** `框架组件.md` 页面骨架模板中的 `:root` 声明（保持一致）
3. **各组件 README.md 和 example.html 无需逐个修改**——它们通过 `var(--token-name)` 引用 token，AI 生成代码时会自动从本文件获取最新值
4. 如需新增 token，在 `:root` 中添加变量，然后在对应组件 README 中引用即可

## 八、图片填充规范

生成含图片的组件（商品卡、酒店房型卡、图片组件等）时，**必须使用 Unsplash 真实图片填充**，禁止使用纯色占位符或空白区域。

### URL 模板

```
https://images.unsplash.com/photo-{ID}?w={宽度}&h={高度}&fit=crop
```

参数说明：`w` 宽度 px、`h` 高度 px、`fit=crop` 裁剪适配。可追加 `&q=80` 提升画质。

### 使用规则

1. **根据内容语境选图**：AI 根据卡片的实际业务场景（目的地、酒店、景点、美食等），从下方图片库中选择最匹配的 ID
2. **尺寸按组件规范调整**：在 URL 中按组件要求设置 `w` 和 `h`（@2x 值），如商品卡图片 `w=140&h=210`，酒店主图 `w=690&h=300`
3. **同一页面不重复**：同一页面内多张图片应使用不同 ID，避免视觉重复
4. **CSS 中使用 `background-image`**：`background-size: cover; background-position: center;` 并保留 `background-color: #E0E0E0` 作为加载兜底
