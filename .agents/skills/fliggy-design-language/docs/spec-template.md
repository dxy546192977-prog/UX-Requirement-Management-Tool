# 组件 spec 规范结构（`spec.md`）

平台 / 业务组件描述文件建议统一为下列章节（无内容可写「无」或删除小节）。

---

## 元信息

- **slug**：`...`（与目录名一致）  
- **分类**：平台通用 / 业务 — `...`  
- **依赖**：`foundations/design-foundations.md`；可选依赖其他组件 slug  
- **最后同步**：YYYY-MM-DD  

---

## 概述

1～3 句说明组件在 App 中的用途。

---

## 使用场景

- 何时使用 / 何时不用（列表）。

---

## 视觉规范（@2x px）

- **尺寸**：宽高、内边距、与栅格关系（6 的倍数、页面边距 18px 等）。  
- **字体**：层级对应 foundations 字号表（H1～Caption）。  
- **颜色 / 圆角**：**仅**通过 `var(--color-*)`、`var(--radius-*)` 引用；若需新 token，先在 foundations 登记。  

---

## 结构与层次

- DOM 层级大纲或示意（可列表缩进）。  
- 关键 class / data 属性命名约定（BEM 或项目前缀）。

---

## 状态与变体

- 默认 / 悬停 / 禁用 / 选中 等；多皮肤写清变体名。

---

## 交互与动效（可选）

- 点击、滑动、长按等；若有动画，写时长与曲线原则。

---

## 可访问性（可选）

- 焦点、朗读标签、最小点击区域等。

---

## 产出物

- **spec.md**：本文件。  
- **example.html**：单文件可运行示意（内联样式须已 token 化或与 spec 一致）。
