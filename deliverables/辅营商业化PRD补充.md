# 交通辅营商业化 PRD 补充

> 建议追加到现有语雀文档末尾，作为“产品化落地方案”章节。原文已有商业模式、目标和进展，本补充重点补齐产品范围、链路规则、数据口径、验收标准与风险。

# 一、产品定位与目标

## 1.1 产品定位

交通辅营商业化不是单点广告位售卖，而是围绕“确定性出行意图”的商业化产品体系。产品需要把机场&文旅补贴、线上电商 CPS/曝光、线下门店导流三类商业机会，沉淀为可配置、可投放、可归因、可结算、可复盘的标准能力。

核心产品价值：

+ **对用户：**在购票、行前、行中场景中获得与目的地、出行人群、行程阶段相关的权益、商品或服务，形成“同价多得”的获得感。
+ **对商家/广告主：**获得比泛流量更确定的目的地、时间、消费能力与出行场景人群，提升曝光、到店、核销、留资或成交效率。
+ **对飞猪：**在不损伤机票主流程转化的前提下，形成补贴提毛利、广告曝光、CPS 跟佣、商品溢价、线下导流佣金等多元收入。

## 1.2 北极星目标

+ FY27 财年交通辅营商业化收入目标：**1.5 亿**。
+ 结构拆解：
    - 机场&文旅局商业化：7000 万。
    - 线上电商商业化：5000 万。
    - 线上引流线下商业化：3000 万。

## 1.3 产品目标

+ 将三类商业化模式从项目制运营升级为标准化产品能力，减少每个客户/每场活动的定制开发成本。
+ 支持按城市、机场、航线、目的地、出行日期、人群标签、订单阶段进行精准投放。
+ 打通曝光、点击、领券、跳转、加购、支付、核销、留资、结算的全链路数据。
+ 建立商业化资源位的体验保护机制，保障搜索、比价、下单等机票核心链路效率。

# 二、用户与业务角色

| 角色 | 核心诉求 | 产品需要支持 |
| --- | --- | --- |
| 出行用户 | 获得与行程相关的优惠、权益、服务，避免被无关广告打扰 | 精准推荐、权益清晰、价格透明、跳转顺畅、可取消/可关闭 |
| 机场/文旅局 | 拉动航线客流、目的地曝光、区域文旅消费 | 补贴策略配置、活动页承接、投放效果报表、资金消耗复盘 |
| 淘宝天猫/异业商家 | 触达高价值出行人群，提升成交或 CPS 跟佣 | 人群定向、资源位投放、商品/权益承接、归因与结算 |
| 线下商户 | 获取到店核销、留资、销售线索 | 优惠券发放、门店承接、核销回传、留资表单、佣金结算 |
| 商业化 BD | 快速包装售卖方案，明确资源库存与效果承诺 | 标准资源包、刊例/价格、投放排期、客户维度报表 |
| 交通运营 | 控制主流程体验与商业化效率 | 白名单、频控、排序规则、实验能力、异常监控 |
| 财务/结算 | 明确收入确认、分润、成本与毛利 | 结算单、核销单、佣金口径、对账明细 |

# 三、需求范围与优先级

## 3.1 P0 范围：6-7 月跑通闭环

+ 支持三类商业化活动的基础配置：机场文旅补贴、线上电商 CPS/曝光、线下券/留资。
+ 支持核心资源位投放：机票 Listing、OTA 结果页、下单页、支付成功页、订单详情页、活动承接页。
+ 支持基础人群与场景定向：出发/到达城市、机场、航线 OD、出行日期、舱位、乘机人数、会员等级、订单阶段。
+ 支持用户转化链路：曝光、点击、领券、跳转淘宝/天猫或商家页、加购溢价包、提交留资、线下核销。
+ 支持活动维度数据看板：曝光、点击、CTR、领券、核销、留资、成交、佣金、收入、成本、毛利。
+ 支持基本风控：资源位频控、活动预算上限、库存上限、素材审核、异常下线。

## 3.2 P1 范围：8-9 月规模化

+ 建设商业化活动配置后台，支持运营低代码创建活动、绑定资源位、选择人群策略、配置预算和投放周期。
+ 建设资源包管理，沉淀“目的地曝光包”“高价值人群 CPS 包”“文旅节气营销包”“线下到店留资包”等标准售卖产品。
+ 支持 A/B 实验与 holdout 对照，评估商业化模块对机票主流程转化率、支付率、客诉率的影响。
+ 支持自动化归因与结算：按曝光、点击、订单、核销、留资等不同模式生成对账明细。

## 3.3 P2 范围：FY27 持续建设

+ 商家自助投放与素材提报。
+ 商业化库存预估与排期锁量。
+ AI 人群/场景推荐，根据 OD、目的地、出行时间、同行人、价格敏感度推荐最优权益或广告主。
+ 多行业模板化方案：汽车、免税、3C、户外、母婴、美妆、体检、地产等。

## 3.4 暂不纳入

+ 不在首期建设完全自助广告平台。
+ 不承诺对外开放用户个人身份明细，只提供合规聚合报表。
+ 不在首期覆盖酒店、度假、火车票等全交通旅行链路，优先聚焦机票核心链路。

# 四、整体产品链路

```text
客户/商机录入
  -> 活动与权益配置
  -> 人群与场景匹配
  -> 资源位投放
  -> 用户转化
  -> 归因与核销
  -> 对账结算
  -> 复盘与策略迭代
```

## 4.1 活动配置

每个商业化活动至少包含以下信息：

+ 活动类型：机场文旅补贴、营销全案、曝光广告、CPS 跟佣、溢价商品、线下券、留资收佣。
+ 合作方信息：客户名称、行业、合同编号、投放负责人、结算方式。
+ 投放周期：开始时间、结束时间、日预算、总预算、库存上限。
+ 定向条件：城市、机场、OD、目的地、出行日期、舱位、人群、订单阶段。
+ 资源位：Listing、OTA、下单页、支付成功页、订单详情页、活动页、IM/消息触达。
+ 转化目标：曝光、点击、领券、跳转、加购、支付、核销、留资。
+ 结算口径：CPM、CPC、CPS、CPA、固定营销费用、补贴节省、溢价收入。

## 4.2 投放决策

+ 优先匹配用户当前行程相关的权益或广告主，避免泛化投放。
+ 同一用户同一订单链路内，商业化模块需做频控，避免多资源位重复展示同一客户。
+ 同资源位存在多个活动时，排序因子建议为：相关性 > 合同优先级 > 收入预估 > 库存消耗进度 > 用户体验保护。
+ 当活动预算、库存、素材、承接页任一异常时，资源位自动降级为空态或平台自营权益。

## 4.3 转化与归因

+ 曝光/点击类：按资源位、活动、客户、OD、人群维度记录。
+ CPS 类：记录飞猪侧点击与外部成交回传，约定归因窗口，避免跨渠道争议。
+ 溢价商品类：记录商品包展示、勾选、支付、退款、售后状态，按实付差价或约定分成计入收入。
+ 线下券类：记录领券、到店核销、核销金额、返佣金额。
+ 留资类：必须记录用户授权状态、表单提交、线索有效性回传、佣金确认状态。

# 五、分场景产品需求

## 5.1 机场&文旅局商业化

### 5.1.1 机票补贴提毛利

+ 支持按机场、航线、城市、出行日期配置补贴池。
+ 支持补贴资金的预算消耗监控，展示“预算总额、已消耗、剩余额度、预估节省平台补贴”。
+ 支持补贴规则与机票价格表达联动，在满足机场/文旅要求的前提下，按运营策略控制补贴强度。
+ 支持按活动维度计算 UE：票量、交易用户、补贴成本、平台节省成本、毛利提升。

### 5.1.2 旅行24季营销 IP

+ 支持按节气/目的地创建营销活动页，承接目的地介绍、航线推荐、权益领取、直播/线下活动入口。
+ 支持活动页挂载异业广告主资源，例如车企、3C、家电、户外、美妆等。
+ 支持线上曝光与线下活动数据合并复盘：站内曝光、点击、报名/留资、线下到场、媒体曝光。

### 5.1.3 异业整合招商

+ 支持将机场/文旅活动资源包装为标准招商资源包。
+ 资源包字段建议包含：主题、城市/目的地、预计曝光、核心人群、资源位清单、刊例价、可售档期、交付物。
+ 支持同一活动下多个广告主并行投放，但需控制同屏数量与行业冲突。

## 5.2 线上电商商业化

### 5.2.1 曝光广告

+ Listing/OTA 资源位应围绕用户当前目的地、出行时间、同行人特征展示相关权益或商品。
+ 资源位表达需以权益/服务价值为主，不应干扰机票价格比较和购票决策。
+ 支持 CPM/CPC 两种基础售卖口径，并保留客户维度的曝光、点击、CTR、eCPM、eCPC 报表。

### 5.2.2 CPS 跟佣

+ 支持跳转淘宝/天猫/商家承接页，并透传活动 ID、资源位 ID、OD、目的地、用户分层等归因参数。
+ 支持按商家/类目/目的地配置不同推荐策略，例如免税、租赁、3C、户外、美妆、亲子用品。
+ 支持佣金回传后的订单匹配、佣金确认、退款剔除与月度对账。

### 5.2.3 溢价商品

+ 支持“机票 + X”商品包，例如机票 + 免税券、机票 + 租赁权益、机票 + 目的地消费礼包。
+ 在下单页展示时必须明确商品包价格、权益内容、有效期、退款规则。
+ 支持用户主动勾选/取消，不默认强制加购。
+ 支持按订单维度记录溢价包收入、退款、履约状态。

## 5.3 线上引流线下商业化

### 5.3.1 海外购物优惠券后返

+ 支持按国际目的地配置商家券包，例如韩国、泰国、马来西亚、香港等。
+ 支持领券后在订单详情页、行前提醒、目的地页进行二次触达。
+ 支持通过支付宝核销或商户回传完成核销闭环。
+ 支持按核销金额、核销用户数、返佣比例计算佣金收入。

### 5.3.2 线下门店曝光留资

+ 支持在高价值人群场景展示留资入口，例如超经、两舱、商旅、家庭出行。
+ 留资表单需明确告知合作方、用途、隐私授权与后续联系规则。
+ 支持线索状态回传：已提交、有效、无效、成交、结算。
+ 支持按有效线索或成交线索计佣。

# 六、核心产品能力

## 6.1 商业化活动管理

+ 新建/编辑/暂停/下线活动。
+ 配置合作方、投放周期、预算、资源位、人群策略、素材、承接页。
+ 支持活动复制，提升同类目的地或同类客户复用效率。
+ 活动状态包括：草稿、待审核、待投放、投放中、已暂停、已结束、异常下线。

## 6.2 素材与权益管理

+ 素材类型：图片、标题、卖点、按钮文案、落地页链接、券/商品 ID、留资表单 ID。
+ 素材审核维度：合规、行业资质、价格真实性、权益有效期、承接页可用性。
+ 权益状态需实时校验：库存不足、已过期、不可领、不可核销时自动隐藏或降级。

## 6.3 人群与场景策略

+ 基础条件：出发城市、到达城市、机场、OD、国际/国内、出行日期、预订提前期。
+ 订单条件：搜索中、比价中、下单中、支付成功、行前、行中、行后。
+ 人群条件：会员等级、高 V/88VIP、商旅、家庭、多人同行、价格敏感、目的地偏好。
+ 频控策略：单用户单日、单订单、单资源位、单客户维度均需可配置。

## 6.4 资源位管理

+ 资源位需维护库存、优先级、可售状态、样式规范和体验红线。
+ 资源位需支持灰度与 A/B 实验。
+ 核心购票链路资源位必须支持快速关闭，避免影响机票交易。

## 6.5 数据看板与结算

+ 支持按活动、客户、资源位、城市、OD、人群、日期拆解数据。
+ 支持日报/周报导出，满足 BD 对客复盘和财务对账。
+ 结算报表需包含：计费模式、计费单价、有效量、无效量、收入、成本、毛利、确认状态。

## 7.3 数据口径

+ 曝光：资源位进入可视区域且停留达到约定阈值后计一次曝光。
+ 点击：用户主动点击商业化模块、权益按钮、商品包入口或留资入口。
+ 有效线索：以合作方回传或双方约定规则确认为准，需剔除重复、无效号码、用户撤回授权等情况。
+ CPS 成交：以外部商家回传成交并过退款周期后的订单为有效成交。
+ 溢价收入：以用户实际支付的商品包收入扣除退款、补偿、履约成本后的口径计算。
+ 补贴节省：对比原平台补贴预算与机场/文旅资金覆盖后的实际平台投入计算。

# 八、体验与风控要求

## 8.1 体验红线

+ 商业化模块不得遮挡价格、航班时间、退改签、行李额等关键购票信息。
+ 下单页溢价商品不得默认强制勾选，需用户主动选择。
+ 同一页面商业化模块数量需受控，避免信息噪声。
+ 商业化素材需明确权益来源、有效期、使用条件和退款规则。
+ 外跳链路需有明确承接，不允许跳转到无关页面或无法返回。

## 8.2 合规要求

+ 留资场景必须取得用户明确授权，并展示合作方、用途和隐私说明。
+ 定向投放只使用合规可用的人群与场景标签，不对外输出用户明细。
+ 医美、金融、地产、酒旅高风险行业需走单独素材和资质审核。
+ 海外购物、免税、线下券需确认地区政策、核销规则、发票与税务口径。

## 8.3 异常处理

+ 预算耗尽：停止投放，资源位降级为平台自营权益或隐藏。
+ 库存不足：停止发券/加购，展示替代权益。
+ 外部回传失败：进入补偿队列，保留点击与交易日志，支持人工对账。
+ 客诉升高：按活动、资源位或客户维度快速下线。
+ 主流程转化下降超过阈值：实验自动报警，运营确认后降权或暂停。

# 九、验收标准

## 9.1 P0 功能验收

+ 运营可以创建至少三类活动：机场文旅补贴、线上电商 CPS/曝光、线下券/留资。
+ 至少 5 个核心资源位可接入商业化活动：Listing、OTA、下单页、支付成功页、订单详情页。
+ 每个资源位支持按城市、机场、OD、出行日期、订单阶段配置投放规则。
+ 用户可以完成至少 4 类转化：点击跳转、领券、加购支付、留资提交。
+ 数据看板可以按活动查看曝光、点击、转化、收入、成本、毛利。
+ 活动支持暂停和下线，异常权益不会继续展示。

## 9.2 业务验收

+ 6-7 月完成 25 个机场/文旅标的签约支持，收入标的达到 2000 万。
+ 6-7 月落地 4 个全案营销活动，引入异业广告收入 300 万。
+ 6-7 月线上电商协议标的达 500 万，异业广告协议达 800 万。
+ 6-7 月线上电商综合收入达到 1000 万。
+ 6-7 月海外购物场景协议标的达到 1000 万，产生佣金收入 500 万。
+ 6-7 月线下门店曝光留资收佣达到 100 万。

## 9.3 体验验收

+ 接入商业化模块后，机票主流程支付转化率不得出现不可解释的显著下降。
+ 商业化模块页面加载失败不影响机票搜索、下单、支付。
+ 客诉率、模块关闭率、退款纠纷需在运营可接受阈值内。
+ 所有留资、外跳、加购、核销场景具备清晰的用户告知。

# 十一、风险与待确认

## 11.1 主要风险

+ **主流程转化风险：**商业化模块可能影响用户比价和下单效率，需要通过灰度、A/B、频控和快速下线机制保护交易。
+ **收入确认风险：**CPM、CPC、CPS、CPA、固定营销费、补贴节省等口径不同，需要财务提前确认收入确认规则。
+ **归因争议风险：**外部商家 CPS、线下核销、留资有效性存在对账差异，需要约定归因窗口和无效量剔除规则。
+ **履约风险：**券不可用、库存不足、门店不可核销会放大用户投诉，需要投放前校验和投放中监控。
+ **合规风险：**留资、定向、医疗美妆、金融地产等行业需提前完成隐私、资质与素材审核。

## 11.2 待确认问题

+ 三类商业化模式的合同主体、开票方式、收入确认口径。
+ 各资源位的售卖价格、库存预估、优先级与体验红线。
+ 广告素材审核流程由商业化团队、交通运营还是平台治理共同负责。
+ CPS 与外部商家之间的归因参数、回传方式、退款剔除周期。
+ 线下券核销是否统一走支付宝能力，还是允许商户自有核销系统回传。
+ 留资场景的数据留存周期、合作方使用范围、用户撤回授权处理。
+ FY27 目标拆到季度、月份、行业、客户后的业务 owner 与交付节奏。
