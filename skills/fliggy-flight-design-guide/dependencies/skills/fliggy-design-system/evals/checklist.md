# 分享前检查清单

## 命名

- skill 发布标识是否为 `fliggy-design-system`
- 目录名是否使用统一 kebab-case

## 规则完整性

- `SKILL.md` 是否写清楚触发范围、非目标、场景分流、输出要求
- `references/` 是否区分普通页面、AI 页面、Figma 来源和设计语言收敛方式
- `examples/` 是否既有日常 prompts 也有回归测试 prompts

## 页面质量

- 是否严格限定为飞猪移动端 UI 页面
- 是否有统一版心和 375 逻辑宽意识
- 是否包含默认、加载、错误、禁用等状态
- 是否保证主 CTA 清晰可见
- 是否避免通用 AI 审美模板和固定骨架
- 页面需要图片时，是否使用真实且业务相关的图片而不是占位图
- 是否写清楚 `aone-kit` 依赖、安装命令和检查命令

## AI 页面

- 是否先判断是否需要 IM 对话型
- 是否避免把 AI 页面做成普通页面换皮
- 是否确保 AI 回答流有“结论 → 理由 → 下一步”

## 下单页

- 是否有订单摘要、表单、价格明细、支付方式、确认按钮
- 是否有字段级校验和错误提示
- 是否避免按钮与底部固定区重叠
- 是否避免把详情页大图头部误用到下单页
- 是否避免为了装饰给下单页硬塞无关大图
- 普通主按钮是否黄底黑字，预订/立即抢/支付是否红底白字

## Figma 校准

- 如果用户给了 Figma 链接，是否先判断是组件、实例还是 AI page
- 如果已有真实页面截图，是否优先以截图和实例校准骨架

## 图片与安全

- 如果接入图片接口，是否通过运行时配置读取 API key
- 是否避免把用户提供的图片 API key 写进 skill 文件、代码仓库或提交记录
- 图片搜索词是否优先使用实体词 + 业务词组合，而不是泛场景词
