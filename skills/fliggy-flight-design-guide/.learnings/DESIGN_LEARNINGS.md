# Design Learnings

飞猪机票设计 Agent 的设计失败经验记录。每次用户反馈"不通过"并完成参考-再生成循环后自动记录。

## 分类标签说明

| 标签 | 含义 | 典型场景 |
|------|------|---------|
| `visual` | 视觉问题 | 字重不够、颜色对比度低、图标风格不统一 |
| `layout` | 布局问题 | 间距过大、对齐偏移、模块顺序不合理 |
| `interaction` | 交互问题 | 缺少 hover 状态、点击区域太小、动效不自然 |
| `data` | 数据展示问题 | 价格格式不对、时间展示不清晰、信息层级混乱 |
| `style` | 风格偏好 | 整体调性不符、过于花哨或过于朴素 |

## 晋升规则

同一 category 的条目累计达到 3 条以上时，自动提炼为设计规律并晋升：
- 视觉/布局规律 -> 对应模块的 spec.md "设计细节备忘" 章节
- 通用风格偏好 -> SKILL.md 的 "生成规则" 章节
- 交互模式 -> 对应 patterns/*.md

晋升后原条目状态更新为 `promoted`，标注晋升目标路径。
