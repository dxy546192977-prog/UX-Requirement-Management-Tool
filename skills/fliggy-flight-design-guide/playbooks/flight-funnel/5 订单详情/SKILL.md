# Fliggy Order Detail Skill

飞猪订单详情页的移动端 UI 设计规范，适用于航旅订单查看、出票后履约、行程展开、乘机人信息确认、订单补充信息展示，以及底部问答支持区的页面生成。

> **读取策略（重要）**：本文件是唯一入口。**禁止一次性读取全部模块文件。** 正确流程：读完本文件 → 识别订单结构 → 读取 [框架组件.md](框架组件.md) → 仅按需读取对应输出组件 `README.md` → 仅在结构不明确时再读取 `example.html`。

## 一、技术基准

- 输出格式：单文件 HTML + 内联 CSS（`<style>` 标签）
- 目标平台：移动端（App WebView / H5）
- 设计基准：750px 宽移动端画板，所有标注均按 @2x px 直接输出
- 页面总高参考：3111px
- 页面主背景：`#F2F3F5`
- 模块主间距：18px；履约区内部主间距：24px
- 组件圆角以 4 / 6 / 12 / 30 / 42 为主，不擅自引入新体系
- 默认中文字体：`PingFang SC`
- 时间、金额、飞行时长等数字优先使用 `Fliggy Sans 102`
- `<head>` 必须包含：
  ```html
  <meta name="viewport" content="width=750, user-scalable=no">
  ```

## 三、最新页面拆分（按设计稿命名）

这版页面拆成 7 个一级区块，生成时应遵循这个顺序和命名：

1. **1 履约区**
   - `1-1 背景`
   - `1-2 导航栏`
   - `1-3 订单状态`
   - `1-4 按钮`
   - `1-5 履约卡`
2. **2 行程区**
   - `2-1 价格、订单栏`
   - `2-2 单程`
3. **2.5 海免商品推荐**（条件展示：出发/到达为三亚或海口 & 已支付 & 未退款）
4. **3 出行信息**
5. **4 订单信息**
6. **5 用研**
7. **6 常见问题**
8. **7 客服入口**

任何生成都不要打乱这个大顺序。

## 四、需求分析（每次生成前执行）

收到订单详情类需求后，按以下顺序判断：

1. **订单状态**：出票成功 / 航变 / 待支付 / 已完成 / 改签中 / 退款中
2. **履约动作**：值机、选座、退改签、产品说明、行李额、出行必读
3. **行程结构**：单程 / 往返 / 中转；是否有航变标；是否需要展开时间轴
4. **信息密度**：是否展示乘机人、证件、票号、航司编码、联系手机、邮箱
5. **尾部支持信息**：是否展示里程领取、用研数字条、常见问题胶囊、联系飞猪

分析结果格式：

```text
【订单状态】出票成功 / 航变 / 已完成
【页面骨架】履约区 -> 行程区 -> 出行信息 -> 订单信息 -> 用研 -> 常见问题 -> 客服入口
【核心动作】值机 / 复制编码 / 展开行程 / 发送行程 / 领取里程
【需要读取的组件】从组件索引中列出
```

## 六、页面编排策略

### 1. 第一屏只做“状态 + 动作 + 履约”

履约区是首屏最重要的信息块。必须优先让用户看到：

- 当前订单是否安全、是否要立刻处理
- 可以立即点击的动作
- 与出行当天最相关的履约卡

### 2. 行程区是信息核心区

总额、订单号、退改签链接、段程卡、时间轴、中转说明必须视为一个连续阅读区，不要拆散。

### 3. 出行信息区要保持稳定列表感

乘机人、航司编码、手机、邮箱、发送行程都属于同一类“出行准备信息”，必须沿用统一左右列列表，不要混成卡片瀑布流。

### 4. 用研与常见问题分成两段

`5 用研` 和 `6 常见问题` 是两个模块，不要再合并成一个大 FAQ 区：

- 用研：渐变头 + 引导标签 + 数字快捷条
- 常见问题：白底标题 + 两行问题胶囊

### 5. 客服入口单独收尾

`7 客服入口` 独立成页尾，不与常见问题混排。

## 七、输出规范

- 所有样式必须写在 `<style>` 标签内
- 颜色、圆角优先使用 token；仅少量透明边框可保留原值
- 模块尺寸、内边距、排列方式优先服从设计稿，不做响应式自由发挥
- 胶囊按钮默认不加阴影
- 履约区的黄色渐变必须保留
- 顶部导航 tab 必须体现“订单详情”选中态
- 金额、时间、时长优先使用 `Fliggy Sans 102`
- 用研数字条必须保持“黄底 + 白圆数字泡”的节奏

## 九、适用与不适用场景

### 适用

- 机票订单详情页
- 航变订单详情页
- 出票成功页
- 订单信息核对页
- 基于飞猪视觉还原静态 HTML 的任务

### 不适用

- 搜索推荐页
- 下单表单页
- 对话式 AI 聊天页
- 酒店详情页
- 旅游攻略页
