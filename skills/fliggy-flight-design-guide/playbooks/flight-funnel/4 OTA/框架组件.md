# 框架组件

国内机票 OTA 结果页的整页框架规范。职责只包括页面壳层、固定导航、滚动容器、模块装配顺序，以及各模块之间的拼接关系。

---

## 一、整页壳层

```text
页面宽度: 750px
页面视窗高度: 1624px
页面背景: var(--color-page-bg)
滚动方式: 导航固定，内容区滚动
body: overflow hidden
```

### 结构职责

- `body` 只承载页面，不负责纵向滚动
- 根容器固定为 `750 × 1624`
- 顶部导航高度固定 `176`
- 下方滚动区高度固定 `1448`
- 根目录 `example.html` 必须体现真实页面壳，不保留阴影、演示圆角和额外外边距

### 组合约束

1. 不允许出现第二个外层滚动容器。
2. 不允许把整页改成普通长文档流。
3. 商品模块和心智模块必须位于滚动区内。

---

## 二、导航栏

```text
总高度: 176px
组成: 状态栏 88px + 导航栏 88px
背景: var(--color-surface)
定位: 固定在页面壳顶部
```

### 默认可见态

- 左侧返回箭头
- 中部航线组：`杭州 + 导航栏单程icon + 北京`
- 右侧功能位：`低价提醒 / 我的收藏 / 更多`
- 状态栏右侧使用 134×24 切图
- 航线组必须按 OTA 原稿做绝对居中，不允许因为左右功能位宽度不同而发生视觉偏移

### 资产要求

- 返回：48×48
- 导航栏单程icon：42×42
- 提醒：36×36
- 收藏：36×36
- 更多：48×48

### 与下游模块关系

- 导航结束后，滚动区从 `176px` 处开始
- 行程卡紧贴导航下边界，不加额外白边

## 九、组合约束

1. 框架组件不写商品卡内部细节，商品卡细节全部下沉到 `输出组件/商品卡片`。
2. `出行提醒` 属于行程卡默认态，不单独拆目录。
3. 整页输出时，`导航栏固定 + 内容区滚动` 是强制规则。
4. 单模块示例可以脱离整页壳层，但必须保持与整页一致的默认态和资产。
5. 导航栏标题必须保留中间航线 icon，不允许使用 `— / -> / →` 等文本占位。
