# 内容标准（格式要求与内容标准）

## 1. SKILL.md 必须包含

1. Skill 职责（做什么，不做什么）
2. 触发场景（什么时候必须加载）
3. 执行步骤（阶段化步骤）
4. 输出标准（可验收清单）

## 2. SKILL.md 建议结构

- 元信息头（name/version/description）
- 标题
- 职责
- 触发场景
- 执行步骤
- 输出标准
- 推荐命令
- 安装方式

## 3. README.md 必须包含

- 适用场景
- 目录结构
- 快速开始
- 安装方式

## 4. 索引文档规范（skills/README.md）

必须包含以下块：

- 技能一览表（目录、定位、触发语、适用时机）
- 决策树（按任务场景选择 Skill）
- 依赖边界说明（`.agents/skills/` 与 `skills/` 的分工）
- 维护约定（新增/修改规则）

## 5. 文案规范

- 中文为主，路径与命令使用反引号。
- 尽量使用可执行语言，避免空泛描述。
- 标准文档中避免与项目实现细节强绑定。
