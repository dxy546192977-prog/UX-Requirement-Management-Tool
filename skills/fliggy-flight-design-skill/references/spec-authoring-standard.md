# 模块 `spec.md` 写法标准

机票页面模块的 `spec.md` 建议包含以下结构（与 `references/agent-protocol.md` 中「spec 写法标准」一致）：

## 1. 这是什么？

一段话说明模块功能、在页面中的位置、与上下游模块关系。

## 2. 如何配置它？

- 配置项表格：名称、类型、可选值、默认值、是否必填。
- 与 `example-full.html` / `example.html` 中占位注释的对应关系。

## 3. 一些例子

给出 1～3 个「改配置 → 期望视觉效果」的简短示例（可含代码片段）。

## 4. 设计细节备忘

易踩坑点：动效、空态、错误态、埋点、实验开关等。

## 5. 外部资源清单

切图 URL、字体 CDN、SVG 路径、依赖的远程接口说明。
