# 内容结构与改写标准

## 1) 改写边界

- 用户明确“只改文本”时：
  - 允许：`TEXT` 节点内容
  - 禁止：布局、颜色、尺寸、间距、图层结构、组件状态

## 2) 内容一致性规则

- query 一致：所有文案围绕同一用户问题
- heroFeature 一致：例如“全网最低价优先”必须贯穿推荐、CTA、浮层
- 链路一致：上一步选择，必须在下一步被明确承接

## 3) 推荐卡与浮层联动规则（本次确认）

- 当用户选择方案一（极致低价）：
  - 方案 CTA 要体现“查看全网比价”
  - 浮层标题要体现“已选方案一”
  - 浮层说明要写明“同航班全网实时价格，按总价升序”
  - 价格列表按低到高排序

## 4) 文案质量基线

- 信息可核验：时间、航司、舱位、价格、渠道数量口径一致
- 单句清晰：避免一条文案承载多个动作
- 行为导向：CTA 必须与下一步行为一致

## 5) 交付口径标准

交付内容至少包含：

1. 改动范围（页面/模块）
2. 关键节点 ID 与文案变更摘要
3. 验证结果（截图）
4. “仅改文本，未改 UI”声明（如果任务边界如此）
